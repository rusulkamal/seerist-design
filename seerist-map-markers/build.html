Phase 1

Content Type

Shape only — no icons, colour, or border variation. Establish the rendering baseline: can all four marker types be placed on MapLibre as overlay divs with correct sizing, positioning, and overlap behaviour at every zoom level?

Event
News & Social
Analysis
Asset
Build
  • Render all marker shapes on the map in a single neutral slate colour
  • Circle (34 px) for events
  • Rounded square with dotted border for News & Social
  • Rounded square with soft green fill for Analysis
  • Filled circle for Assets
  • Cluster bubble for grouped events
Feasibility
  • Do overlay divs stay correctly anchored to map coordinates during pan and zoom?
  • Do shapes render crisp at 34 px on retina / high-DPI displays?
  • At what marker count does frame rate start to drop?
  • Does the CSS dotted border render consistently across Chrome, Safari, and Firefox?
Phase 2

Event Status

Add border patterns and badges to events. Verify that CSS dashed borders on circular elements behave consistently across browsers, and that absolutely-positioned badges stay anchored during map movement.

  • Dashed border + bolt badge (top-right) + pulse ring → Breaking
  • Stagger pulse animation across markers (0.28 s offset per marker)
Feasibility
  • Does CSS dashed border on a circular element render consistently across browsers?
  • Do absolutely-positioned badges stay anchored to markers during zoom and pan?
  • Can the pulse ring CSS animation run on 20+ simultaneous markers without CPU/GPU impact?
  • Do staggered pulse delays (0.28 s offsets) hold up correctly in practice?
Phase 3

Category

Phase 4

Severity

Add colour as the final layer. Verify that colour values propagate correctly through the currentColor pattern and that the system degrades gracefully when colour is unavailable.

  • Does the system degrade meaningfully if colour is stripped (forced grayscale / accessibility mode)?
