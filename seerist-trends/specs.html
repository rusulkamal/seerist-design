Layout
Split panel
Trend tab is a fixed side-by-side layout — results list on the left, charts on the right. Both panels scroll independently.
List panel 340px fixed width · independently scrollable · always shows results filtered by all active chart selections
Chart panel flex: 1 remaining width · independently scrollable · 5 chart cards stacked with 32px gap
List count bar Sticky at top of list panel. Default: N results. When chart filters active: text turns green and reads N of Total results match.

Mode switcher — Distribution vs Over Time
Each chart card has a mode switcher and a sub-type switcher in its header.
Distribution Aggregated counts per value across the full filtered date range. Sub-types: Pie · Bar
Over Time Counts per value broken into daily or weekly buckets. Sub-types: Bar · Line · Stacked Area
On mode change Sub-type resets to default for that mode (Distribution → Pie, Over Time → Bar). Active filter for that chart is preserved — does not clear on mode switch.
On sub-type change Chart re-renders with animation. Filter state and selected series are preserved.
Over Time mode maps data to 7 fixed weekly buckets. If a result's weekIndex falls outside the current date range filter, it will still be bucketed but won't appear in activeResults — so counts may be 0 for those weeks. This is expected behaviour, not a bug.
Collapse / expand
Each chart card can be collapsed to just its header row.
Trigger Chevron button in top-right of card header. Toggles .collapsed class on the card.
Collapsed state Chart body, legend, filter chips, and meta row are hidden via CSS. Chart instance is not destroyed — it stays in memory and keeps updating.
Filter state Active filters on a collapsed chart remain active and continue to affect other charts and the results list.
Click-to-filter
Selecting a value on a chart
Clicking any data element on a chart adds or removes that value from the chart's filter set.
1
User clicks a data element Works on: pie slice, distribution bar segment, over-time bar segment, line/area data point. The clickable element is resolved via getElementsAtEventForMode('nearest', {intersect: true}).
2
Label is resolved Pie / distribution bar → label from visible series by index. Over-time bar/line/area → label from visible series by datasetIndex. Hidden series are excluded from the visible array before resolving.
3
Filter set is toggled Each chart maintains its own Set of active filter values. If the label is in the set → remove it. If not → add it. Multiple values can be active simultaneously (OR logic within the same chart).
4
All visible charts refresh Each chart calls getFilteredFor(ownKey) to recompute its data. The results list and count bar also update.
Hover tooltip
Hovering a data element shows a floating tooltip with label and count before the user commits to a click.
Position Fixed position following the cursor: clientX + 14px, clientY − 38px. Single shared tooltip instance for all charts.
Distribution content Label · count · percentage of total · "click to filter"
Over Time content Label · count for that week · week label · "click to filter"
Cursor Changes to pointer on hover, resets to default when no element is under cursor.

Cross-chart filtering
How filters combine across charts
Selecting a value on Chart A narrows the data shown in Charts B, C, D, E — but not Chart A itself.
Logic AND across charts — a result must match the active selection in every chart to appear. OR within a chart — if multiple values are selected on the same chart, a result matching any of them passes.
Self-exclusion rule When computing data for Chart A, getFilteredFor('A') applies all charts' filters except Chart A's own filter. This prevents Chart A from filtering itself to zero when a selection is active.
Results list Uses getFilteredFor(null) — all charts' filters applied, no exclusions. Strictest possible intersection.
Context indicator When Chart B is being filtered by Chart A, Chart B's header shows a blue chip: Emotions · Anger. Shows chart name + selected values. Truncates to first 2 values + +N more when more than 2 are active.
Spike badge Shown in chart header when any series has moved ≥10% in the most recent week vs the previous 2 weeks. Computed after cross-filter is applied, so it reflects the currently filtered data.
Filter chip row
Appears below a chart's header when that chart has at least one active selection.
Visibility Hidden by default. Shown when activeFilter.size > 0 for that chart.
Per-chip remove Each chip has an × button. Clicking it removes only that value from the filter set and triggers a full refresh.
Clear all "Clear all" link appears when activeFilter.size ≥ 2. Clears the entire filter set for that chart only.
Global clear "Clear filters" button in the top bar clears filter sets across all charts simultaneously. Shows a toast: "Filters cleared — Showing all results".
Legend — show / hide series
Legend chips
Legend sits below the chart body. Each chip represents one series. Click to hide; click again to show.
Hidden state Chip goes to opacity: 0.35 with dashed border. Dot goes grey. Series is removed from chart data and the visible series array.
Filter interaction Hiding a series that is currently in the active filter set removes it from the filter set. A hidden series cannot be filtered by. This prevents a filter referring to a value no longer visible in the chart.
Index resolution Click targets on the chart are resolved against the visible series only array, not the full series list. If series 0, 1 are hidden and the user clicks the first visible segment, the resolved index is 0 of the visible array — not 2 of the full array.
Restore Clicking a hidden chip restores it to the full series array and triggers a full refresh.
Global filters
Date · Location · Source
Top-bar filters narrow the base result set before any chart filtering is applied. Changing any global filter resets all chart selections.
FilterOptionsEffect
Date range Last 7 / 14 / 30 / 90 days Sets a minimum weekIndex. Results with a lower index are excluded from activeResults.
Location All · Middle East · Europe · Asia Pacific · Americas · Africa Filters activeResults to items where region === selectedValue. "All" disables the filter.
Source All · News · Social Media · News + Social "News + Social" passes a comma-separated string; the filter splits on comma and uses Array.includes(). "All" disables the filter.
On any change 1. Clear all chart filter sets. 2. Recompute activeResults. 3. Animate each chart body to opacity: 0.4, then after 350ms remove loading class and refresh. 4. If chart filters were active before the change, show toast: "Chart selections reset — Search changed, charts updated to match".

Special cases
Selecting the same value twice
Clicking a value that is already selected deselects it (toggle behaviour).
Behaviour The value is removed from the chart's filter Set. If this was the only active filter on that chart, the chart returns to its full-data state (no dimming). All other charts refresh accordingly.
Multiple values selected on the same chart
A user can select more than one value on a single chart.
Filter logic OR — a result passes if its field value matches any of the selected values for that chart. For example: selecting "Anger" and "Fear" on Emotions shows all results that are Anger or Fear.
Visual Both segments display in highlight colour. All unselected segments stay at reduced opacity. Filter chip row shows one chip per selected value.
Filtering while in Over Time mode
Over Time mode shows series as separate datasets. Click targets resolve via datasetIndex, not index.
Click target For bar/line/area, the selected label comes from visibleSeries[els[0].datasetIndex]. Clicking a specific week's bar selects the series (e.g. "Sadness"), not the week. The week position is used in the tooltip only.
After selection Selected dataset renders at full weight (borderWidth: 2.5, full point radius). Others render at reduced opacity. Chart does not switch back to Distribution mode.
Global filter change while chart filters are active
Changing date range, location, or source resets the entire chart filter state.
Reset scope All 5 chart filter sets are cleared simultaneously. The user's mode and sub-type choices per chart are preserved — only the filter selections are reset.
Toast Only shown if filters were actually active at the time of the global change. Not shown on a clean state change.
Hiding a series that is currently filtered
Legend hide and active filter interact: hiding removes the value from the filter set.
Why If a hidden series stayed in the filter set, other charts would continue to show data filtered by a value that is invisible and inaccessible. This would be confusing and unrecoverable without a "clear all".
Sequence 1. Add label to hidden set. 2. Remove label from filter set if present. 3. Trigger full refresh. This means the cross-chart filter effect of that value disappears instantly.
Edge cases
All series hidden on a chart
Scenario
User hides every series in the legend on a single chart.
Behaviour
Chart renders with no visible data. The chart canvas remains visible but empty. No crash — vis array is empty so datasets/labels are empty arrays. Other charts are unaffected. No empty state shown — this is intentional for this phase.
Cross-filters produce 0 matching results
Scenario
Combining filters from multiple charts results in zero items matching all conditions simultaneously.
Behaviour
Results list shows 0 cards with no message (count bar shows 0 of N results match). Each chart still renders — with getFilteredFor(key) each chart still sees some data because it excludes its own filter. The chart showing the last filter that caused the zero match will still have data for itself.
Global location filter + Region chart filter both active
Scenario
User sets Location dropdown to "Europe" and also clicks "Middle East" on the Region chart.
Behaviour
Global filter runs first on activeResults — only Europe results remain. The Region chart then tries to filter by Middle East, but no Middle East results exist in activeResults. Result list shows 0 items. Region chart shows only Europe data (it excludes its own filter when rendering). This combination is valid but confusing — a future improvement could warn when a chart filter contradicts a global filter.
Spike badge with insufficient data
Scenario
Date range set to "Last 7 days" — only the most recent week bucket exists. No previous weeks to compare against.
Behaviour
computeSpike() checks if both recent and prev arrays are non-empty before computing. If either is empty, returns null. The spike badge is simply not rendered. No error, no fallback label.
Chart in collapsed state while filters change
Scenario
User collapses a chart card, then makes a filter selection on another chart.
Behaviour
The collapsed chart's instance is still updated via refreshChart(). Its data is correct when the user re-expands. The chart does not visually animate the loading state while collapsed — the canvas is hidden so the opacity fade has no visible effect.
Tooltip persists on rapid tab switch
Scenario
User hovers over a chart element, then quickly switches to the List tab before the mouseleave event fires.
Behaviour
The tooltip is a fixed-position div that is hidden on mouseleave. Switching tabs hides the chart panel but the tooltip element remains visible until the next hideTT() call. Fix recommendation: call hideTT() in switchTab() to prevent orphaned tooltips.
Switching mode with an active filter
Scenario
User selects "Anger" on the Emotions pie chart, then switches to Over Time mode on the same chart.
Behaviour
The filter for "Anger" remains in the set. In Over Time mode, the Anger dataset renders at full weight and all others are dimmed. The filter chip row still shows the "Anger" chip. This is correct and intentional — the user's selection persists across mode changes.
Source filter "News + Social" combined with Source chart filter
Scenario
User selects "News + Social" from the global source dropdown, then clicks "Blog" on the Source Type chart.
Behaviour
Global filter removes Blog and Report from activeResults. Clicking "Blog" on the chart adds it to the chart filter set, but since Blog results don't exist in activeResults, the filtered result list is empty. Source Type chart still shows News and Social bars (it excludes its own filter). Same "conflicting filters" scenario as the region case above.
